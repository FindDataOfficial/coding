## Refined Structure — 3 Sub-Goals

### Sub-goal 1: MVP (CLI-usable platform)

1. **Design the YAML server registry** — Data model for registered MCP servers (name, source path, runtime config, port, health URL, status). Stored in a YAML file matching the existing `default.yaml` registry pattern.
2. **Build the subprocess lifecycle manager** — Python module: start (subprocess/uvicorn), stop (graceful shutdown), restart, status check. Port allocation in range 8100-8199. Subprocess only for MVP.
3. **Implement nginx reverse proxy + discovery** — nginx routes each server to its assigned port. `/api/servers` registry endpoint for discovery. SSE transport support (disable buffering, set proper headers).

### Sub-goal 2: Dashboard + Monitoring

4. **Add monitoring & health checks** — Per-server health endpoint polling, log collection (stdout/stderr capture), basic metrics (uptime, request count, last error). Expose via `/api/servers/:id/health` and `/api/servers/:id/logs`.
5. **Build the web dashboard** — Extend existing dashboard pattern: list registered servers, deploy new ones, start/stop/restart, view logs and health. Vanilla JS + Tailwind CDN, no build step.

### Sub-goal 3: Docker + Tests + Docs

6. **Add Docker deployment support** — Generate Dockerfile, build, run. Graceful degrade if Docker not installed.
7. **Write tests** — pytest integration tests for lifecycle manager (start/stop/restart), routing (endpoint reachability), API tests.
8. **Document end-to-end workflow** — `code/create-mcp` generates server → PAAS deploys → dashboard monitors.

## Key Decisions

- **nginx config**: Auto-generated by the platform when servers are registered
- **Sample MCP server**: A minimal hand-written server will be used for MVP development; `code/create-mcp` is a separate concern
- **SSE through nginx**: Requires `proxy_buffering off`, `proxy_cache off`, and appropriate `Content-Type` / `Cache-Control` headers

## Original Success Criteria (still apply)

- Deploy an MCP server with one CLI command or dashboard click, reachable at its endpoint
- Start/stop/restart works without orphaned processes
- Dashboard shows real-time status of all servers
- Health checks detect down servers within 30 seconds
- Docker mode: `docker-compose up` equivalent launches the server containerized
- All tests pass

## Constraints (unchanged)

- Local-first: works on a single machine without cloud dependencies
- Docker support is optional (gracefully degrade if Docker not installed)
- Dashboard follows existing dashboard pattern (vanilla JS, no bundler)
- Generated servers should be deployable with zero manual config changes
- Port range: 8100-8199 (configurable)
